# Layer 패키지 - 개발 규칙

## 레이어 격리 원칙 (절대 위반 금지)

### 레이어는 서로의 존재를 알아서는 안 된다
- **레이어 코드(`.kt`) 및 KDoc에 다른 레이어명·함수명을 절대 언급하지 않는다**
- 예: `FileSystemLayer`의 KDoc에 `llm.analyze` 언급 금지
- 예: `LLMLayer`의 KDoc에 `file-system.readFile → llm.analyze → file-system.writeFile` 패턴 언급 금지
- 이 규칙을 어기면 레이어 간 강결합이 생겨 독립적 교체·추가가 불가능해진다

### 레이어는 자신의 책임만 설명한다
- KDoc은 "이 레이어/함수가 무엇을 하는가"만 설명한다
- "언제 이 레이어를 다른 레이어와 함께 써야 하는가"는 레이어가 결정하지 않는다
- 조합 패턴은 LLM이 각 레이어의 설명을 읽고 스스로 결정한다

### 올바른 KDoc 작성법
```
// ✅ 올바름 — 자신의 동작만 설명
/**
 * 파일 읽기, 쓰기, 탐색을 담당하는 레이어
 * - readFile: 파일 내용을 읽어 반환
 * - 변환 결과를 저장해야 하면 writeFile을 children에 연결하세요
 */

// ❌ 잘못됨 — 타 레이어 언급
/**
 * - readFile 결과를 llm.analyze로 넘겨 분석 후 writeFile로 저장하세요
 */
```

### 레이어 추가/수정 체크리스트
- [ ] 신규 레이어 KDoc에 타 레이어명이 포함되어 있지 않은가?
- [ ] 기존 레이어 KDoc이 신규 레이어를 참조하도록 수정하지 않았는가?
- [ ] `execute()` 분기에서 다른 레이어를 직접 호출하지 않는가?
